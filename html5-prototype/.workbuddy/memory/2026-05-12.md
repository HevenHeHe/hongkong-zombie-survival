# 2026-05-12 工作日志

## Bug 检测与修复 - Hong Kong Zombie Survival HTML5 Prototype
对 `index.html` 进行了全面代码审查，发现并修复了 6 个 bug：

### 修复清单
1. **Bug 2 (🔴严重)**: 僵尸速度 `0.02~0.04` × 50 后仅 1-2 单位/秒，追不上玩家(速度5) → 改为 speed `3.5~6.0` 直接使用，不再乘50
2. **Bug 6/7 (🔴严重)**: 玩家和僵尸物理体未设置 `angularDamping`，碰撞后翻滚 → 玩家加 `angularDamping=1, linearDamping=0.1`，僵尸加 `angularDamping=0.95`
3. **Bug 8 (🟠中)**: `pointerdown` 在触屏时与摇杆冲突误触攻击 → 增加 `event.pointerType === 'touch'` 过滤
4. **Bug 3 (🟠中)**: 僵尸掉落世界(y<-10)被当击杀加分(score+10) → 移除掉落加分，只做清理
5. **Bug 4 (🟠中)**: fruit采集后先全量入库再吃1个导致库存堆积无出口 →改为fruit直接全部吃掉恢复饥饿，不入库
6. **Bug 1 (🟡轻)**: `restartGame()` 调用 `startGame()` 结构冗余 → restartGame 中显式调用 `clearSurvivalObjects()` 再调 startGame

## 额外修复：游戏启动失败
- **根因**: `cannon-es@0.20.0` 从 CDN 以 `<script>` 标签加载时报错 `Uncaught SyntaxError: Unexpected token 'export'`，因为 0.20+ 只提供 ES Module 格式，不支持 UMD 全局变量方式加载
- **错误链**: cannon-es 加载失败 → `CANNON is not defined` → `setupCannon()` 崩溃 → `init()` 崩溃 → 整个游戏白屏
- **修复**: 将 CDN 源从 `cdn.jsdelivr.net/npm/cannon-es@0.20.0/dist/cannon-es.js` 切换到 `unpkg.com/cannon-es@0.19.0/dist/cannon.min.js`（0.19.0 是最后一个支持 UMD/全局变量的版本）
- 同时给两个 CDN script 标签加了 onerror 错误提示，方便未来排查类似问题
